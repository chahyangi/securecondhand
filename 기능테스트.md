# 기능 테스트 체크리스트

`중고거래플랫폼_설계문서.md` 7.1절(완료된 기능) 기준으로 작성한 수동 테스트 체크리스트다. 두 계정(A, B) 이상으로 브라우저 탭을 나눠 띄우고 진행하면 채팅/친구/대리인/대면 거래 인증까지 확인할 수 있다. `- [ ]`를 `- [x]`로 바꿔가며 체크할 것.

---

## 1. 회원/계정
- [ ] 회원가입(`/signup`) — 아이디/비밀번호/닉네임 입력 후 가입, 자동 로그인되는지
- [ ] 로그아웃 후 같은 계정으로 로그인 성공
- [ ] 잘못된 비밀번호로 로그인 시도 → 에러 메시지 표시
- [ ] 새로고침 후에도 로그인 상태 유지 (sessionStorage 토큰)
- [ ] 같은 브라우저의 다른 탭에서 다른 계정으로 로그인해도 서로 토큰이 안 섞이는지
- [ ] 설정 › 계정 관리에서 비밀번호 변경 (현재 비밀번호 확인 필요한지)
- [ ] 비밀번호 변경 후 기존 세션이 로그아웃되고 재로그인해야 하는지
- [ ] 설정 › 계정 관리에서 회원 탈퇴 → 탈퇴 후 로그인 불가 확인

## 2. 상품 등록/조회/수정/삭제
- [ ] 상품 등록 — 사진 여러 장 업로드, 제목/카테고리/상태/거래방식/가격/설명 입력
- [ ] 가격을 0 이하 또는 빈 값으로 등록 시도 → 프론트에서 막히는지
- [ ] 등록한 상품이 메인 그리드에 정사각형 썸네일로 나타나는지
- [ ] 상품 상세에서 여러 장 사진이 스와이프/슬라이드로 보이는지
- [ ] 본인 상품 수정(`/product/:id/edit`) — 사진 삭제, 순서 변경(대표 사진 지정), 텍스트 수정
- [ ] 본인 상품 삭제
- [ ] 판매중 → 예약중 → 판매완료 상태 전환, 배지가 바뀌는지
- [ ] 판매완료 상품은 "채팅하기" 버튼이 비활성/문구 변경되는지
- [ ] **다른 사람 계정으로 로그인해서 남의 상품을 수정/삭제/상태변경 시도 → 403/에러로 막히는지 (권한 검증)**
- [ ] 메인 그리드 스크롤 끝까지 내려서 다음 페이지가 자동으로 이어붙는지 (무한 스크롤)
- [ ] 검색창에서 키워드/카테고리/거래방식/정렬 필터 적용

## 3. 찜 / 신고
- [ ] 상품 상세에서 찜 버튼 토글 → 새로고침해도 유지되는지
- [ ] 설정 › 찜 목록에서 찜한 상품 조회 및 개별 해제
- [ ] 상품/사용자 신고 — 사유 입력 후 접수
- [ ] 설정 › 신고 내역에서 본인이 신고한 것만 보이는지 (남의 신고 내역 안 보이는지)
- [x] 서로 다른 계정 3개로 같은 상품 신고 → 목록/상세에서 사라지는지(자동 차단) — curl로 확인(`보안점검.md` 5장)
- [x] 서로 다른 계정 3개로 같은 유저 신고 → 그 유저 로그인이 막히는지(자동 휴면) — curl로 확인
- [x] 같은 계정 하나로 반복 신고해도 차단되지 않는지 (신고 어뷰징 방지) — curl로 확인
- [ ] (UI) 관리자 대시보드 › 상품 탭에서 자동 차단된 상품에 "자동차단됨" 표시가 보이는지

## 4. 친구 관리
- [ ] 아이디로 사용자 검색
- [ ] 친구 요청 보내기
- [ ] 상대 계정에서 받은 요청함 확인 → 수락/거절
- [ ] **요청을 보낸 사람이 자기 요청을 스스로 수락할 수 없는지 (받은 사람만 가능해야 함)**
- [ ] 수락된 친구 목록에 반영되는지
- [ ] 친구 삭제 / 보낸 요청 취소

## 5. 채팅 (실시간)
- [ ] 하단 네비 "채팅" → 채팅방 목록(`/chats`)에 참여 중인 방이 상대방/상품/거래상태와 함께 보이는지
- [ ] 상품 상세에서 "채팅하기" → 채팅방 생성/입장 (같은 상품으로 재입장 시 기존 방 재사용되는지)
- [ ] 텍스트 메시지가 양쪽 브라우저 탭에 실시간으로 보이는지
- [ ] 새로고침해도 메시지 히스토리가 유지되는지
- [ ] 채팅 중 사진 전송 → 상대방 화면에 즉시 렌더링되는지
- [ ] 친구 중 1명을 대리인으로 초대 → 채팅방이 3인으로 확장되는지, 시스템 메시지가 뜨는지
- [ ] 대리인이 아닌 사람은 초대 버튼이 안 보이는지

## 6. 대면 거래 인증
- [ ] 판매자→(대리인)→구매자 순서로 각 구간 버튼을 눌러야 다음 구간이 열리는지 (순서 위반 시 막히는지)
- [ ] **구매자 계정으로 판매자 인증 버튼을 대신 누르려는 시도 → 403으로 거부되는지 (인증 위조 방지)**
- [ ] 모든 구간 완료 시 채팅방 상태가 자동으로 "거래확정"으로 바뀌는지
- [ ] 새로고침해도 인증 진행 상태가 유지되는지
- [ ] 대리인 포함 거래 완료 시 대리인이 자동으로 채팅방에서 퇴장되는지

## 7. 알림 설정
- [ ] 설정 › 알림 설정에서 채팅/거래 알림 토글 변경
- [ ] 새로고침 후에도 토글 상태가 유지되는지

## 8. 관리자
- [ ] 일반 계정으로 `/admin` 접근 시 차단되는지
- [ ] `is_staff` 계정으로 `/admin` 접근 → 통계, 전체 사용자/상품/신고 목록이 보이는지
- [ ] 사용자 활성/정지 토글
- [ ] 상품 강제 삭제
- [ ] 신고 처리완료 처리

## 9. 환경/배포 확인
- [ ] `.env` 없이 저장소를 새로 clone했을 때 `README.md` 안내만으로 로컬 실행이 되는지
- [ ] `npm run build`가 에러 없이 끝나는지 (Node 20+ 환경에서)
- [ ] `python manage.py check` / `migrate`가 경고 없이 끝나는지

## 10. 송금 (과제 최소 요구사항 — 구현 완료, `개발과정.md` 2.5절 · `보안점검.md` 4장)
아래 중 API 레벨(curl)로는 이미 검증됨(`보안점검.md` 4장). 브라우저 UI로도 두 계정(A, B)을 띄워 직접 확인할 것.
- [x] 잔액 충전(또는 초기 지급) 후 다른 유저에게 송금 성공 — curl로 확인(회원가입 시 기본 1,000,000원 지급)
- [x] 잔액보다 큰 금액 송금 시도 → 거부 — curl로 확인
- [x] 0원/음수 금액 송금 시도 → 거부 — curl로 확인
- [x] 본인에게 송금 시도 → 거부 — curl로 확인
- [x] 동시에 같은 잔액으로 여러 번 송금 요청(동시성) → 잔액이 음수로 내려가지 않는지 — curl 동시요청 10건으로 확인
- [ ] (UI) 설정 › 송금 화면에서 아이디로 송금 → 잔액/내역 화면에 즉시 반영되는지
- [ ] (UI) 채팅방의 "송금하기" 버튼으로 구매자→판매자 송금 → 채팅방에 시스템 메시지로 표시되는지
- [ ] (UI) 송금 내역이 보낸 사람/받은 사람 화면에 각각 -금액/+금액으로 올바르게 표시되는지

## 11. 지갑 충전 — 토스페이먼츠 테스트 연동 (`개발과정.md` 2.7절 · `보안점검.md` 6장)
아래 중 curl로 이미 검증된 것과, 브라우저에서 실제로 확인해야 하는 것을 구분해뒀다. 사전에 `backend/.env`에 `TOSS_CLIENT_KEY`/`TOSS_SECRET_KEY`가 설정되어 있어야 한다.
- [x] 주문 생성 → 승인 요청 금액을 주문 금액과 다르게 보내면 거부되는지 — curl로 확인
- [x] 가짜 payment_key로 승인 시도 → 토스 서버가 거부하고 주문이 실패 처리되는지 — curl로 확인
- [x] 이미 승인된 주문을 다시 승인 요청해도 잔액이 중복 적립되지 않는지 — curl(시뮬레이션)로 확인
- [ ] **(UI, 미검증) 설정 › 송금 › "충전하기" → 금액 입력 → 결제하기 → 토스 테스트 결제창이 실제로 뜨는지**
- [ ] **(UI, 미검증) 토스 문서의 테스트 카드번호로 결제 완료 → `/wallet/topup/result`로 리다이렉트 → "충전이 완료됐어요" 표시 및 잔액에 실제 반영되는지**
- [ ] (UI) 결제창에서 취소/실패 시 실패 메시지가 뜨고 잔액이 그대로인지
